## 使用流程

1. 浏览器打开永久链接
2. ① 配置：添加面试官、添加候选人（选岗位自动带出对应 STAR 问卷与评分权重）
3. ② 评分：选面试官+候选人 → 整张结构化面试表内联展示 → 逐维度 1-5 分评分 + STAR 记录 → 提交
4. ③ 结果：自动按权重加权汇总（综合分 = Σ维度平均分×权重 / Σ权重），≥4.0 建议录用 / 3.0-3.99 待定 / <3.0 不建议

## 一票否决与破格录用模块（面试自由填写）

评分表单末尾含两个可填写模块，内容由面试官按岗位/候选人自行填写，系统**不预置任何具体项**（如驾照、证件、英语等）：

- **🚫 硬性指标一票否决**：点「＋添加否决项」逐条填写门槛（如：无大电流PCB经验 / 有诚信问题），每项选「达标/不达标」。任一项不达标 → 结果直接判「一票否决·不建议」，无论综合分多高。
- **⭐ 特别优秀破格录用**：填写「破格维度」+「成就证据」。有证据且综合分非否决时 → 结果标记「破格录用」，由 HR 结合多人意见最终确认。

判定优先级：**一票否决 > STAR 加权综合分 > 破格录用**。

## 面试官按岗位分配 / 删除 · 候选人归档（历史不丢）

- **面试官按岗位分配**：在「① 配置 → 管理岗位」每个岗位下方，勾选该岗位对应的面试官；新增面试官默认进入全部岗位，可在具体岗位下取消勾选。不同岗位可分配不同面试官。
- **面试官可删除**：点面试官名旁的 ✕ 即移除；已提交的评分记录**保留**（以"署名归档"形式留在历史中），仅从当前名单与岗位分配中移除。
- **候选人"归档"而非删除**：点候选人「归档」后，其全部评分记录保留、结果页仍可查，仅移出当前名单；归档区提供「恢复」可随时放回。
- **评分页只显示本岗位面试官**：进入「② 评分」选候选人后，面试官下拉仅列出该岗位已分配的面试官，避免串岗。

> 设计原则：任何"删除"都不破坏已产生的面试记录，保证历史可追溯、可审计。

## 多人协作（两种方式，推荐云端）

**方式一 · GitHub 云端（推荐，零部署）**：面试官提交即自动上传合并，HR 多端自动同步，无需来回传链接。详见下方「GitHub 当云端」。

**方式二 · 离线链接（无云端时）**：
- HR 点「复制协作链接」→ 微信发面试官
- 面试官打开评分，点「复制我的评分链接」回传
- HR 打开回传链接 → 自动合并 → 结果页汇总
- 链接过长被截断时，改用「导出配置 / 导出评分」JSON 文件经微信发送，再「导入并合并」

## GitHub 当云端（零部署 · 推荐）

用你自己的 GitHub 仓库当云端数据库，**免注册任何第三方平台**：

1. 生成一个 **GitHub 细粒度 Personal Access Token（Fine-grained PAT）**：仅勾选本仓库 `xuyao7695/hr-interview-tool` 的 `Contents: Read/Write`，有效期设到本轮招聘结束即可。
2. ① 配置 → 「GitHub 云端自动合并」填写：
   - 仓库：`xuyao7695/hr-interview-tool`
   - 云端文件名：自定义，如 `dk2026.json`
   - GitHub Token：上面的细粒度令牌
3. 点「保存云端设置」→ **地址栏会自动带上云端配置**（`?gh=…&ghf=…&ghk=…`）；也可点「⬆ 上传配置到云端」先把云端文件建好。
4. 发面试官：点「复制面试官链接」→ 其打开**仅见评分页**，提交即自动写回 GitHub。
5. 汇总：HR 在结果页点「☁ 从云端刷新」即合并全部面试官提交。

### 跨设备自动同步（本次修复重点）

> 之前的问题：HR 主链接是裸地址，在新手机/电脑（无本地缓存）打开时不会拉取云端 → 数据不同步。
> 现已修复：**HR 的"主链接"本身就携带云端配置**，换设备打开自动同步。

- **HR 换设备**：保存云端设置后，**直接复制浏览器地址栏**（已含 `gh/ghf/ghk`）或点「🔗 复制主链接」按钮，在手机/电脑打开 → 自动从 GitHub 拉取**全部数据**（岗位 / 面试官 / 候选人 / 评分 / 归档 / 云端配置），无需重新填写仓库信息。
- **面试官换设备**：「复制面试官链接」发出的链接已含云端配置，任何设备打开即同步。
- 链接中的 Token 仅用于读写该云端文件，用后可在 GitHub 撤销（见下方安全事项）。

## 技术要点

- 纯前端单文件（index.html），零后端依赖，永久部署于 GitHub Pages
- 数据默认存浏览器 localStorage；开启 GitHub 云端后，仓库文件为唯一真源，多端自动同步
- 协作链接通过 URL hash（Base64）编码实现离线合并；云端模式通过 GitHub Contents API 读写仓库文件

## 本次更新（按使用反馈重构）

- **跨设备自动同步**：HR 主链接携带云端配置，换手机/电脑打开自动拉取全部数据；新增「🔗 复制主链接」按钮。
- **面试官专属链接**：「复制面试官链接」生成 `?role=interviewer` 链接，面试官打开后**仅见评分页**；可在链接里带 `&iv=<id>` 预选面试官并隐藏选择器。
- **面试官按岗位负责**：新增面试官**不再自动分配全部岗位**，在「管理岗位」下按岗位勾选负责面试官即可。
- **评分页按岗位过滤**：选候选人后面试官下拉仅显示该岗位负责者；切换候选人时下拉随之刷新，支持同一链接连续多场切换。
- **归档 / 硬删**：配置页候选人生「归档」（移出名单、隐藏于面试官下拉，但结果页保留历史、可恢复）与「删除」（彻底移除本人及评分）；结果页每条记录带「🗑 删除」（仅 HR 可见）。
- **格式化报告**：结果页「📄 导出报告」生成含每位面试官评分/评语/最终结果的报告，可打印或存 PDF 转发。
- **自动录用建议**：综合结论由系统按"一票否决 > 破格 > 加权"自动判定，面试官无需手动选"建议/待定/不建议"。
- **维度权重可手工调整**：预置岗位各维度权重可在「管理岗位」下直接修改并保存（合计需 100%）。

## 安全事项（重要）

- 云端用的 **GitHub 细粒度 PAT 用后请到 https://github.com/settings/tokens 撤销**（建议有效期设到本轮招聘结束）。
- 撤销不影响已部署页面，仅影响后续"上传/刷新云端"权限；撤销前请确认数据已落盘或已导出报告。
- 含 Token 的链接等同云端读写凭据，**仅发给参与本轮面试的面试官**；若链接外泄，及时撤销 Token 即可。
